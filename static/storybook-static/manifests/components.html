# EasyBuy Storybook

> A React Native Web Storybook for the real EasyBuy component surface.

This Storybook is intentionally focused on:

- shared UI used across the auth flow
- auth-facing composition pieces
- full auth screen assemblies where layout and copy matter
- states that are easier to review visually than in code alone

## Current coverage

| Area | Why it starts here |
| --- | --- |
| Shared inputs, feedback, and actions | High reuse and easy visual review |
| Auth layout, headings, and headers | Defines the feel of the current implemented scope |
| Config-driven auth fields | Shows how form config maps to real field rendering |
| Full auth screen forms | Captures composition, spacing, and copy in one place |

## System note

This Storybook is part of the larger EasyBuy docs system:

- Docusaurus explains the system
- Storybook shows component states
- TypeDoc documents exported contracts

As the docs grow, this surface should stay curated around authored components and real screen compositions rather than turning into a dump of every generated primitive.
