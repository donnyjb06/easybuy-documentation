## What it does

`PasswordInput` renders a controlled password field with the same validation shell as `TextInput`, plus a show and hide toggle.

## Why it exists

Password fields appear in login, registration, and recovery flows. Keeping the toggle and validation behavior in one place prevents those flows from drifting.

## Example

```tsx
type ResetValues = {
  password: string;
};

<PasswordInput<ResetValues>
  control={control}
  name="password"
  label="New Password"
  placeHolder="Enter a new password"
/>
```

## Props

| Prop | Type | Notes |
| --- | --- | --- |
| `control` | `Control<T>` | React Hook Form control for the current form. |
| `name` | `FieldPath<T>` | Field name registered in the form state. |
| `label` | `string` | Upper label rendered above the field. |
| `placeHolder` | `string` | Placeholder text for the password field. |
