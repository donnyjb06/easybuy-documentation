## Type Families

| Family | Examples | Why it matters |
| --- | --- | --- |
| Context and state | `AuthContextType`, `AuthData`, `AuthResponse` | Defines the auth boundary and the backend payload shape |
| Schema-inferred form values | `LoginSchemaType`, `RegisterSchemaType`, `VerifyEmailSchemaType` | Keeps the form value docs aligned with the validation source |
| Config shape | `AuthFormInputConfig<T>` | Explains how the config-driven auth fields are assembled |
| External types | `RelativePathString`, `FieldPath`, `Control`, `z.infer` | Should be linked or named, not redefined in local docs |
